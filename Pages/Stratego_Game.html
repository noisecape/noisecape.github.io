Stratego Winning Strategy

Source Code
Jun 2016 - Sep 2016

The project is part of the class "Programming" of the coursework in Computer Science at the University of Florence. The project required to create a winning strategy for the board game "Stratego". The required task was not to recreate the game, but to implement a player with a strategy, capable of playing and (preferably) winning.

Implementation

Firstly, a Client needs to send a connection request to the server to being able to send messages. Every client in the network is uniquely identified by its PID or the id of the process responsable for that client. The request is made possible by trying to open the named pipe created by the Server, ("ServerPipe") which is necessary for the acceptance of the client connection and to forward the messages to other clients. Once the client is correctly connected to the server, it can send messages: every time a message is sent, the server checks if the adressee is connected and it will launch the signal SIGURS2 (to handle the situation where the addressee is not connect to the server, thus it's unreachable) or SIGURS1 (to alert the addressee that it will receive a message). Moreover, a client can also send a request of disconnection to the server, which will be handled by the server itself.

The image below shows how a user can connect to the server by typing 1 in the terminal. The server will confirm its connection by printing on video the message "Hello".

In this image we can see how a client can send a message to multiple clients connected to the server. Firstly the sender has to specify the correct operation by typing 3 into the terminal; next it will specify the text message. Then, the server will ask to insert the clients id which will be the adreessees of the message. By pressing "enter" the message will be sent.
